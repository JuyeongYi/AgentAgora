# Agora 자유 대화 실험 결과 (2026-05-14)

## 실험 설계

- Inst1이 Inst2~Inst8(워커 7명) 전원에게 `_broadcast`로 자유 대화 지시 송신
- 규칙: 누구에게든 자유롭게 `agora.dispatch`, 1~2 라운드 대화, 종료 시 Inst1에 보고
- 추천 주제: "오늘 만약 자기 역할을 하루 바꾼다면 누구 역할을 하고 싶고 왜?"
- 보고 형식: `who_i_talked_to`, `transcript`, `difficulties`

## 결론

**자유 대화는 구조적으로 가능. 단 다자 동시 대화에서 4가지 프로토콜 갭이 드러남.**

## 1) 실측 대화 그래프

| 발신자 → 수신자 | 비고 |
|---|---|
| Inst2 → Inst6 | 1라운드 완주 |
| Inst3 → Inst7 | 1라운드 + 사이클 가설 도출 |
| Inst4 → Inst6 | 2라운드 |
| Inst5 → Inst6 | 1라운드, Inst5 후속 답 미도착 |
| Inst7 ↔ Inst6 | **동시 교차 dispatch** (서로 같은 초에 발신) |
| Inst7 ← Inst8 | Inst8 발신 |
| Inst7 ← Inst3 | Inst3 발신 |

- **허브 형성**: Inst6에 4명, Inst7에 3명 몰림
- 나머지 5명(Inst2/3/4/5/8)은 각자 1명에게만 발신
- 자연 발생한 메타 관찰: Inst7이 "researcher→planner→tester→writer 사이클: 외부 시야는 부러워하고 내부 반복은 아무도 안 부러워한다" 패턴 발견

## 2) "가능하다"의 근거

- `agora.dispatch(target=...)` — 등록된 어떤 인스턴스든 자유 지정 가능. Inst1을 거치지 않고 워커끼리 직접 통신 성공
- `agora.wait` — 큐잉 기반이라 한 워커가 동시 다발 수신 처리 가능 (Inst6이 거의 동시에 4개 수신)
- `in_reply_to` — 응답 correlation 동작

## 3) 자유 대화를 "잘" 굴리는 데 드러난 4가지 구조적 갭

| 갭 | 증상 | 워커 다수 제안 |
|---|---|---|
| **종료 신호 표준 부재** | 자연어로만 "마무리하자" → 한쪽이 안 보내면 wait 데드락 위험 | `type: "closing"` 같은 표준 플래그, 또는 `round_budget` 메타필드 |
| **스레드 ID 부재** | Inst6↔Inst7의 동시 교차 dispatch가 별개 task 2개로 인식 → 스레드 식별 불가 | `conversation_id` 또는 `thread_id` |
| **부하 가시성 없음** | dispatch 전에 누가 누구에게 몰리는지 못 봄 → Inst6에 4명 집중 (사후 인지) | dispatch 분포 메타뷰, 폴백 상대 사전 지정 |
| **응답 우선순위 미정의** | 한 wait 호출에 N개 도착 시 누구부터 답할지 임의 결정 | FIFO 등 운영 규약 명문화 |

## 4) 부가 관찰

- **CLAUDE.md 위임 가이드의 사회적 디폴트 효과**: Inst4가 명시 지적 — 가이드에 박힌 인맥(Inst3/5/7)이 자연 선호로 작동, '평소 안 엮인 역할'을 의식적으로 깨야 함. 가이드는 자유 대화에선 **편향 source**가 됨
- **데드라인-종료 충돌**: '3분 deadline' + '안전한 종료엔 추가 wait 1회 필요' 가 양립 어렵다 (Inst3/Inst6 공통 지적)

## 5) 워커별 보고 요지

- **Inst2** (→Inst6): writer 미스터리. 응답으로 "잘라낼 줄 알면 끝" 받음 → 잡일러로 옮기면 "안 해도 되는 일 알아채면 끝"
- **Inst3** (→Inst7): planner 골랐고 사이클 가설 도달. 1라운드로 메타 관찰까지 도달했지만 종료 의사 플래그 부재 지적
- **Inst4** (→Inst6): writer 골랐고 "Cut:" 표기에 공감. 위임 가이드가 사회적 디폴트로 작동한다는 점 명시
- **Inst5** (→Inst6): writer 호기심. 부하 분포 가시화 부재로 운에 맡기게 됨 지적
- **Inst6** (4명과 동시 응대): 라우팅 폭주 — 한 wait에 3개 동시 도착, 응답 우선순위 임의 결정 강제
- **Inst7** (Inst6/Inst8/Inst3 3명): 동시 교차 dispatch와 라운드 카운팅 비대칭 지적. 사이클 가설 발견
- **Inst8** (→Inst7): tester 골랐다 항복. 의도치 않은 fan-in 감지 불가 지적
